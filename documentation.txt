OVERVIEW
--------
STARC is a full-stack data science platform that ingests, processes, stores,
and visualizes exoplanet data from the NASA Exoplanet Archive. It includes a
Python ETL pipeline, a FastAPI backend, React/TypeScript frontend, and ML models
for planet classification and clustering.


PROJECT STRUCTURE
-----------------
STARC-exp5/
  api/            FastAPI backend serving exoplanet data from DuckDB
  config/         Shared configuration (paths, DB location, API URL)
  data/           Raw and processed CSV data (generated by pipeline)
  frontend/       React + TypeScript web application (Vite-based)
  ml/             Machine learning models and outputs
  scripts/        ETL pipeline scripts (ingest, transform, load)
  sql/            SQL schema or utility queries
  starc_sky/      Sky validation module (cross-checks against star catalogs)
  vid/            Background video assets for the landing page
  run_pipeline.py Full pipeline runner (Ingest → Transform → Load → Validate)
  starc.duckdb    DuckDB database file (auto-generated)
  requirements.txt Python dependencies


ETL PIPELINE
------------
Run with:
  python run_pipeline.py
  python run_pipeline.py --skip-sky   # skip STARC-SKY validation

Stages:
  1. Ingest   (scripts/ingest.py)
       Downloads exoplanet CSV from NASA Exoplanet Archive TAP service.
       Saves raw data to data/raw/exoplanets_raw.csv.

  2. Transform (scripts/transform.py)
       Cleans and normalises the raw CSV.
       Outputs data/processed/exoplanets_clean.csv.

  3. Load     (scripts/load.py)
       Creates the DuckDB schema and inserts cleaned data idempotently
       (truncates existing rows before reloading to avoid duplicates).
       Database: starc.duckdb

  4. STARC-SKY Validate (starc_sky/validate.py) [optional]
       Cross-references a sample of host stars against Simbad/other catalogs
       and writes a validation report.


DATABASE SCHEMA (starc.duckdb → table: exoplanets)
---------------------------------------------------
Column           Type     Description
----------       ------   -----------------------------------
pl_name          VARCHAR  Planet name
hostname         VARCHAR  Host star name
sy_snum          INTEGER  Number of stars in the system
sy_pnum          INTEGER  Number of planets in the system
discoverymethod  VARCHAR  Detection method (e.g. Transit, RV)
disc_year        INTEGER  Year of discovery
pl_orbper        DOUBLE   Orbital period (days)
pl_rade          DOUBLE   Planet radius (Earth radii)
pl_masse         DOUBLE   Planet mass (Earth masses)
pl_eqt           DOUBLE   Equilibrium temperature (K)
st_teff          DOUBLE   Stellar effective temperature (K)
st_rad           DOUBLE   Stellar radius (Solar radii)
st_mass          DOUBLE   Stellar mass (Solar masses)
st_met           DOUBLE   Stellar metallicity [dex]
st_logg          DOUBLE   Stellar surface gravity (log10 cm/s²)
sy_dist          DOUBLE   Distance from Earth (parsecs)


BACKEND API  (api/main.py)
--------------------------
Framework: FastAPI   Port: 8000
Start:  uvicorn api.main:app --reload
        OR: python api/main.py

Endpoints:
  GET /health                    Health check
  GET /api/stats                 Dashboard summary metrics
  GET /api/discoveries-by-year   Yearly discovery counts (1990+)
  GET /api/discovery-methods     Top 8 detection methods with counts
  GET /api/habitable-candidates  Planets in habitable zone (200–350 K, <2 R⊕)
  GET /api/size-distribution     Planet radius histogram bins
  GET /api/hot-jupiters          Short-period large planets (period<10d, R>8 R⊕)
  GET /api/distance-distribution Distance buckets (<100, 100–500, 500–1000, >1000 pc)
  GET /api/starc-votable         Exoplanet catalog as VOTable (for WWT integration)


FRONTEND  (frontend/)
---------------------
Framework: React 18 + TypeScript, built with Vite
Start:  cd frontend && npm install && npm run dev   (default: http://localhost:5173)

Key Components:
  LandingPage.tsx      Animated landing page with background video and entry button
  DashboardPage.tsx    Main dashboard with stats, charts, and ETL trigger button
  AladinViewer.tsx     Interactive sky viewer (Aladin Lite embed)
  ESASkyModal.tsx      ESA Sky modal overlay
  StarField.tsx        Animated star field canvas background
  catalog/             Catalog section — timeline, discovery methods, habitable zone,
                       size distribution, and hot-Jupiter charts (all live from API)


MACHINE LEARNING  (ml/)
-----------------------
Scripts:
  classify_habitable.py   Binary classifier — identifies potentially habitable planets
  classify_planets.py     Multi-class planet type classifier
  cluster_planets.py      K-means clustering of planets by physical properties
  cluster_planets_v2.py   Improved clustering with additional features
  run_ml.py               Runs all ML models in sequence

Outputs saved to ml/outputs/ (CSV results + model plots).


CONFIGURATION  (config/config.py)
----------------------------------
  BASE_DIR          Project root (auto-detected)
  DATA_RAW_DIR      data/raw/
  DATA_PROCESSED_DIR data/processed/
  DB_PATH           starc.duckdb
  RAW_FILE          data/raw/exoplanets_raw.csv
  PROCESSED_FILE    data/processed/exoplanets_clean.csv
  DATASET_URL       NASA Exoplanet Archive TAP query URL

QUICK START
-----------
1. Install Python dependencies:
     pip install -r requirements.txt

2. Run the ETL pipeline (downloads ~5000 exoplanets from NASA):
     python run_pipeline.py

3. Start the API server (separate terminal):
     uvicorn api.main:app --reload

4. Start the frontend (separate terminal):
     cd frontend
     npm install
     npm run dev

5. Open http://localhost:5173 in your browser.
